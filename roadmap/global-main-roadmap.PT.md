## 12019 - 12020 - Structure

  

### Drayker initial

  

Desenvolvemos um modelo para especificação formal de colaboração chamado DFM, que será a base para cooperação na construção do Dk e plataforma.

  

**Distributed Function Modeling** - [**DFM**](https://github.com/draykerdk/DFMP)

  

A modelagem de funções distribuídas é um método/protocolo generalista que estrutura dados, eventos e ontologias, quebrando tudo em pequenas funções individuais que são conectadas por junções sinápticas chamadas módulos ou modelagem, que podem se conectar com outros módulos em uma topologia distribuída.

  

### Reference Architecture

As arquiteturas de referência vem sendo desenvolvidas na fase de inteligência, na fase de estruturação as arquiteturas são formalizadas para passarem pelo processo de transição.

  

### Drayker transition

A transição da Drayker e o processo onde passamos por uma abertura ao mundo, é quando começa o processo de recrutamento de voluntários ativando os mecanismos de cooperação em massa.

  
  

## 12020 - 12024 - Engineering Dk 0.1

### Base Structure DK - [BSDK](https://github.com/draykerdk/bsdk)

  

O sistema base do Dk é uma estrutura de dados de montagem, que forma a base para o DFM, sendo a função mãe de toda a cadeia de funções que forma o Dk e a plataforma da Drayker, essa função funciona como as bases de um DNA, que integra todas as funções e módulos de ativação.

  

#### DNA Function Module structure

Estrutura base do kernel distribuído.

  

#### Function structure

Funções iniciais e padrões otimizados.

  

#### Modeling structure

Módulos iniciais e estruturas otimizadas de padronização e tipos.

  

### OSDK VM

OSDK inicial optimizado para máquinas virtuais multiplataforma.

  

#### DNA FM language coupling VM

Compiladores otimizados para execução e programação com a estrutura base BSDK.

  

#### Programming coupling Kernel

Acopladores de kernel multiplataforma, otimização de virtualização para montagem de plataforma dentro de super apps(DKAPPs), base para desenvolvimento de UID.

  

### Dk Network

Módulos iniciais de rede acoplados ao BSDK, bases para computação distribuida.

  

### A.I evolutionary structures

Módulos iniciais de evolução estrutural, otimização, teste, validação e integração evolutivos e automáticos, metaprogramação.

  

### Dknowledge

Base de dados e conhecimento distribuída e modularizada, estruturas funcionais para tipos de dados e níveis de acesso e integração históricos e ligações funcionais para interpretação e transparência de decisões automáticas nas estruturas inteligentes.

  

### Living Cryptography

Integração inicial com criptografia inteligente e identificadores, provas de conhecimento, chaves de ativação e desativação autônomas integradas ao sistema de computação distribuída, formação de uPOW inicial integrado a camada evolutiva e o módulo especialista.

  

## 12023 - 12025 - Test-net

A temporada de teste incentivar o ataque da rede e dos sistemas críticos além de fortalecer as camadas evolutivas com ataques de força bruta e computação massiva para os nodes.

  

### Test Global

Liberação de nodes de teste sem as regras de UID para estudo e evolução da rede em condições extremas.

  

#### Computation

Testes com computação massiva e virtualização de nodes, campanha para participação de datacenters e supercomputadores. Incentivos e formas criativas de conseguir computação massiva para otimização e teste para camada evolutiva e sistema de computação distribuída.

  

#### Secure

Incentivos para ataques bem sucedidos da rede e descoberta de vulnerabilidades imprevistas. Auditorias e testes de segurança automatizados e de força bruta com A.Is.

  

#### A.I modules

Integração e validação de módulos especialistas de A.i na camada de inteligencia. Testes massivos e simulações.

  

#### Cryptography

Revisão e validação matemática de criptografia, simulação e teoria dos jogos.

  

### Global evolutionary corrections

Validação e estudo das correlações evolutivas.

  

### Simulated Test Global Evolutionary

Simulações globais da rede para testar as adaptações evolutivas e forçar adaptações para cenarios simulados.

  

### Global security competition

Competição global de segurança para validação da plataforma 0.1

  

### Structure implementation reference

Documentação e estruturação da implementação de referência e ferramentas para construção da plataforma de implementação.
